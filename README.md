# Smart Cab Allocation System - Backend
A Project for MoveInSync campus hiring

## Overview
The Smart Cab Allocation System backend provides the API and server-side logic for managing cab availability, user interactions, and real-time updates. Built with Node.js, Express, MongoDB, and Redis, it ensures efficient and scalable operations.

## Technologies Used
- **Node.js**: JavaScript runtime for building server-side applications.
- **Express**: A minimal web framework for Node.js.
- **MongoDB**: NoSQL database for storing user and cab data.
- **Redis**: In-memory data structure store for caching.
- **Socket.io**: For real-time communication between clients and the server.
- **dotenv**: For managing environment variables.

## Features
- **User Authentication**: Secure user login and registration.
- **Cab Management**: APIs for searching, updating, and booking cabs.
- **Caching Mechanism**: Utilizes Redis to cache search results for improved performance.
- **Real-time Updates**: Push updates to clients for cab location changes.
